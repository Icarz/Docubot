# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

DocuBot is a RAG (Retrieval Augmented Generation) document assistant built with the Anthropic Claude API. It parses PDFs, chunks and embeds text, stores vectors in ChromaDB, and answers questions grounded in document content with structured JSON responses validated by Pydantic.

## Commands

```bash
# Activate virtual environment (Windows)
.\venv\Scripts\Activate.ps1

# Run the FastAPI server
uvicorn main:app --reload

# Run the RAG pipeline standalone (ingest PDF + query)
python pdf_rag_pipeline.py

# Run prompt evals
python eval_prompts.py

# Run the agent loop (tool-use demo)
python agent_loop_lesson14.py

# Run the agent with conversation memory (SQLite-backed)
python agent_with_memory.py

# Run the interactive chat loops
python chat_loop.py
python chat_loop_v2.py
```

No test framework is configured. `eval_prompts.py` serves as the eval suite ÔÇö it runs 4 test cases against Claude and checks answers with lambda assertions.

## Architecture

### Shared Module

`models.py` ÔÇö single source of truth for `DocumentAnswer` (Pydantic model), `extract_json_from_markdown()`, and `DOCUMENT_QA_SYSTEM_PROMPT`. Imported by all scripts that need structured output.

### Core Pipeline (RAG)

`pdf_parser.py` extracts text from PDFs via `pypdf`, splits into overlapping sentence-based chunks with page metadata. `pdf_rag_pipeline.py` orchestrates the standalone RAG flow: ingest PDF chunks into ChromaDB, retrieve top-k similar chunks, inject as context into Claude, parse structured response.

### API Layer

`main.py` ÔÇö FastAPI backend (v2) with full integration:
- `POST /upload` ÔÇö accepts PDF files, parses and chunks them, stores embeddings in ChromaDB (persistent)
- `POST /chat` ÔÇö retrieves relevant chunks from ChromaDB, injects as context, calls Claude, returns structured answer. Supports multi-turn via SQLite-backed conversation history
- `GET /health` ÔÇö status + collection count
- Uploaded PDFs saved to `./uploads/`, vectors persisted in `./chroma_data/`

### Agent System

`agent_tools.py` ÔÇö basic agent loop (tool_use Ôćĺ execute Ôćĺ repeat).
`agent_loop_lesson14.py` ÔÇö agent loop with `max_iterations` safety limit, three tools: `search_documents`, `lookup_customer`, `calculate_refund`.
`agent_with_memory.py` ÔÇö agent with SQLite-persisted conversation history (`conversations.db`).

### Structured Output Pattern

System prompt demands JSON Ôćĺ response cleaned with `extract_json_from_markdown()` Ôćĺ validated via `DocumentAnswer` Pydantic model (fields: `reasoning`, `answer`, `confidence`, `section_referenced`, `sources`). All defined in `models.py`.

## Key Conventions

- All scripts load API keys via `python-dotenv` from `.env` (must contain `ANTHROPIC_API_KEY`)
- Model used throughout: `claude-sonnet-4-6`
- Temperature `0.2` for all document Q&A calls
- ChromaDB uses `PersistentClient` (data in `./chroma_data/`)
- PDF test fixture: `sample_policy.pdf` (generated by `create_test_pdf.py` using reportlab)

## Dependencies

Defined in `requirements.txt`. Key packages: `anthropic`, `chromadb`, `sentence-transformers`, `pypdf`, `pydantic`, `fastapi`, `uvicorn`, `python-dotenv`, `python-multipart`, `scikit-learn`, `reportlab`.
